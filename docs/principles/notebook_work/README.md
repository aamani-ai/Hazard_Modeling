# Notebook work

Working principles (and, where useful, applied guides) for how we operate inside notebooks — especially
**exploratory data notebooks**, where the job is to *understand* a data source before any modeling uses it.

| File | What it covers |
|------|----------------|
| [`exploratory_data_notebooks.md`](exploratory_data_notebooks.md) | Interpret every variable (value + meaning + reference base + use-decision); understand before using; the checklist of what every exploratory notebook should have. |

> Per the principles [README](../README.md): a principle sub-folder like this may also carry an applied
> **guide** — for hands-on work the *why* and the *how* travel together.
