# docs/

Documentation for **Hazard_modeling**.

| Folder | Contents |
|---|---|
| [`extra/`](extra/) | The scope-and-story anchor + per-session [`tasks_history/`](extra/tasks_history/) handoffs (session→session context). |
| [`google_drive_docs/`](google_drive_docs/) | Local `.docx` copies of the team's shared Google Drive **"InfraSure Hazard"** reference set, with folder links and per-file summaries. **Google Drive is the source of truth.** See [`google_drive_docs/README.md`](google_drive_docs/README.md). |
| [`plans/`](plans/) | Planning docs / plan-of-record: per-phase plans, decisions logs, assumptions registers, imported flood/hurricane review plans, and the active [`hazard_conus_grid`](plans/hazard_conus_grid/) build plan. |
| [`hazards/`](hazards/) | Hazard anchors and cross-hazard modeling guides: source selection, before-M0 fundamentals, and the [`M0-M4 event/loss contract`](hazards/m0_m4_event_loss_contract.md). |
| [`principles/`](principles/README.md) | **The foundational beliefs** ("why we build this way") — standard interface not standard physics · modular from day one · basics spot-on. |
| [`learning_logs/`](learning_logs/README.md) | **What building taught us** — derived knowledge *not in the references*, crystallized so the next peril inherits the lesson. The retrospective counterpart to `principles/` (beliefs) and `plans/` (intent). |
| [`references/`](references/README.md) | **What it's grounded in** — the methodology-provenance map (each method → external source + internal derivation), plus the external [`bibliography`](references/bibliography.md) reproduced in-repo. Answers *"where's this method from / how do you know industry does this?"* |
