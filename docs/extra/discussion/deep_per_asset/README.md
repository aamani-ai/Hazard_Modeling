# docs/extra/discussion/deep_per_asset/ — the deep per-asset product (discussion)

The discussion home for **use case 1**: deep, per-asset hazard risk — a *real* asset at its true location,
the "deep-time" run that produces one trustworthy number. It is the **future second driver** of the shared
risk engine; the CONUS grid (use case 2) is the first, now built. This folder is where we think things
through *before* committing — when to migrate the deep-run notebooks onto the package, and what it takes to
do it right.

| Doc | What |
|---|---|
| [`01_notebooks_to_second_driver.md`](01_notebooks_to_second_driver.md) | When we're ready to shift the deep-run notebooks onto the shared engine, the nuances (collection geometry, frequency model, footprint, QC, validation), the plan shape, and the cross-reference files. |

> Discussion graduates to a plan once decided — the way the grid's `04_production_architecture.md` became
> [`plans/hazard_conus_grid/architecture/`](../../../plans/hazard_conus_grid/architecture/). This is not a
> commitment to build now; it's the map for when we are ready.
